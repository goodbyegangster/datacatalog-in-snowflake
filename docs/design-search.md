# Design Search

各検索向け画面に設置する検索機能について詳細を定義。

## page：データ資産

### 複数の検索カテゴリー間の条件（バケット方式）

- カテゴリー１（フリーワード）は常に AND で効かせる。カテゴリー２〜４はそれぞれに
  `AND` / `OR` を選べるセグメントコントロールを持たせ、次のように結合する。

```text
カテゴリー１ AND ( AND 指定カテゴリーを全て満たす ) AND ( OR 指定カテゴリーのいずれかを満たす )
```

- `AND` を選んだカテゴリーは**全て必須**、`OR` を選んだカテゴリーは**いずれか 1 つ**を満たせばよい。
- 各カテゴリーの初期値は `AND`。
- 未選択（無制約）のカテゴリーは AND 群 / OR 群のいずれにも入れない（結果に影響しない）。
- NOTE: 当初は「カテゴリー間を 1 つのトグルで AND/OR 切替」「各カテゴリー初期は全選択」の
  設計だったが、実装時に「各カテゴリーに AND/OR」「初期は未選択＝無制約」へ変更した。

### カテゴリー１：フリーワード

- 部分一致
- 大文字 / 小文字の区別なし
- 検索対象を以下より指定できるチェックボックスを用意する（初期状態は全てチェック済み）
  1. 「データ資産」の「名前」
  2. 「データ資産」の「説明」
  3. 「カラム」の「名前」
  4. 「カラム」の「説明」
- 「カラム」の名前 / 説明にヒットした場合は、その**所属データ資産**を検索結果に含める
- データ資産一覧には「フリーワード一致」列を表示する
  - 本列は、フリーワード検索により「なぜそのデータ資産が検索結果に含まれたか」を補助表示する列とする
  - フリーワード検索以外（階層 / オブジェクト種別 / タグ）による一致理由は本列には表示しない
  - データ資産の名前 / 説明に hit した場合は `名前 に一致。` / `名前 / 説明 に一致。` のように表示する
  - カラムの名前 / 説明に hit した場合は `カラム名 CUSTOMER_EMAIL に一致。` / `カラム説明 CUSTOMER_EMAIL に一致。` のように実際のカラム名を表示する
  - データ資産自身とカラムの両方に hit した場合は `名前 / 説明 に一致。カラム名 ORDER_ID、カラム説明 ORDER_ID, AMOUNT に一致。` のように文を分ける
  - 表示するカラム名は 2 件までとし、超過分は `ほかN件` と表示する
  - フリーワード由来でない検索の場合は空欄とする
  - 表示例

    | 一致内容 | 表示値 |
    | --- | --- |
    | データ資産名 | `名前 に一致。` |
    | データ資産名・説明 | `名前 / 説明 に一致。` |
    | カラム名 | `カラム名 EMAIL に一致。` |
    | カラム名（複数） | `カラム名 EMAIL, CUSTOMER_EMAIL ほか2件 に一致。` |
    | カラム説明 | `カラム説明 CUSTOMER_EMAIL に一致。` |
    | データ資産名・説明・カラム | `名前 / 説明 に一致。カラム名 ORDER_ID、カラム説明 ORDER_ID, AMOUNT に一致。` |
- 複数文字列での検索
  - OR 条件: 文字列間に ` OR ` が含まれている場合
  - AND 条件: 文字列間に ` AND ` が含まれている場合
  - 両方が含まれる場合は ` OR ` を優先する（片方のみの使用を想定）

### カテゴリー２：階層（データベース / スキーマ）

- データベース
  - 表示対象として読み込まれたデータ資産に含まれるデータベース名を、複数選択可能なプルダウンで表示
    - Snowflake provider では、カタログテーブル読み込み時に `DISPLAY_SCOPES` で事前絞り込み済みの資産を利用する
    - fake provider では、fake provider が返す資産に含まれる値を利用する
  - 初期は未選択（未選択 = このカテゴリーで絞り込まない）
- スキーマ
  - 表示対象として読み込まれたデータ資産に含まれるスキーマ名を、複数選択可能なプルダウンで表示
  - データベース向けプルダウンの子プルダウンとなり、表示される値はデータベース側で選択された値に連動する
  - 初期は未選択
  - データベース向けプルダウンを更新した場合には、自動的に未選択へ更新される
- NOTE: 当初は「初期は全選択」「DB 更新時は全選択へリセット」だったが、
  「未選択 = 無制約」方針に合わせ「初期は未選択」「DB 更新時は未選択へリセット」へ変更した。

### カテゴリー３：オブジェクト種別

- [docs/design-model.md](./design-model.md) の「`ASSETS` テーブル `ASSET_TYPE` 列の求め方」で導出される値を複数選択可能なプルダウンで表示
- 初期は未選択（未選択 = このカテゴリーで絞り込まない）

### カテゴリー４：タグ

- [streamlit/settings.py](../streamlit/settings.py) の `SELECTABLE_TAG_KEYS` で定義されたタグを、タグ毎に複数選択可能なプルダウンで allowed_values を表示
  - allowed_values の値は、カタログテーブル `TAGS` より取得する
  - タグ定義のコメントは、カタログテーブル `TAGS.TAG_COMMENT` より取得し、プルダウン直上の補助説明として表示する
- タグは「データ資産」自身だけでなく、その**いずれかのカラム**に付与されている場合もヒットとみなす
  - 例: `PII` はカラムに付与されることが多い
- 初期は未選択（未選択 = このカテゴリーで絞り込まない）

## page：ユーザー

カテゴリー１にて「ログインユーザーのみ表示」が有効な場合、カテゴリー２を入力不可とする。

### カテゴリー１：ログインユーザーのみ表示

- Streamlit App にログインしている Snowflake ユーザーのみを表示するトグルスイッチを配置
  - ラベルは「ログインユーザーのみ表示」
  - 初期値は OFF（全ユーザー表示）
- トグル ON 時は、ログインユーザーのみを自動的に表示する
- [streamlit/settings.py](../streamlit/settings.py) の `IS_VISIBLE_ONLY_SELF_USER` が True の場合、本トグルは ON 固定となる
- ログインユーザー名を取得できない場合は全ユーザー表示へ fallback せず、一覧を表示しない

### カテゴリー２：フリーワード

- 部分一致
- 大文字 / 小文字の区別なし
- 検索対象を以下より指定できるチェックボックスを用意する（初期状態は全てチェック済み）
  1. 「ユーザー」の「名前」
  2. 「ユーザー」の「表示名」
- 複数文字列での検索
  - OR 条件: 文字列間に ` OR ` が含まれている場合
  - AND 条件: 文字列間に ` AND ` が含まれている場合
